# Managing Courses

Course HTML files are stored in **Supabase Storage** (private bucket), not in git.
The Hugo site only contains auto-generated catalog pages (`content/courses/<slug>.md`).

## Prerequisites

```bash
npm install
cp .env.example .env   # fill in SUPABASE_SERVICE_ROLE_KEY (and Gmail creds for emails)
```

Required `.env` variables for course upload:
- `SUPABASE_URL` — Supabase project URL
- `SUPABASE_SERVICE_ROLE_KEY` — service role key (from Supabase dashboard)
- `SUPABASE_COURSE_BUCKET` — bucket name (default: `course-materials`)

Additional variables for email sending:
- `GMAIL_USER` — Gmail address
- `GMAIL_APP_PASSWORD` — Gmail App Password ([generate here](https://myaccount.google.com/apppasswords))
- `EMAIL_FROM` — sender display name and address

## Directory structure

```
course-source/              # gitignored — local only
  <slug>/
    meta.yaml               # source of truth for file list and course metadata
    lesson-01.html           # full HTML documents (must include <meta charset="UTF-8">)
    lesson-02.html
    ...

content/courses/
  <slug>.md                  # auto-generated by upload script — do not edit by hand

scripts/
  upload-course.js           # uploads files + regenerates catalog
  send-course-email.js       # sends emails to course participants
  meta.example.yaml          # template for new meta.yaml files

email-templates/
  welcome.html               # English welcome email
  welcome-lv.html            # Latvian welcome email
  course-update.html         # new materials notification
```

## Uploading / updating course content

1. Edit or add HTML files in `course-source/<slug>/`
2. Update `course-source/<slug>/meta.yaml` if files were added/removed or metadata changed
3. Run the upload:

```bash
npm run upload-course -- <slug>
```

This will:
- Upload all files listed in `meta.yaml` to Supabase Storage (upserts — safe to re-run)
- Regenerate `content/courses/<slug>.md` from `meta.yaml`

4. Commit and push the regenerated `.md` file:

```bash
git add content/courses/<slug>.md
git commit -m "Update <slug> course catalog"
git push
```

### Important notes

- HTML files must be **full documents** with `<!DOCTYPE html>`, `<meta charset="UTF-8">`, etc.
  Fragments without charset will display broken Latvian characters (mojibake).
- `meta.yaml` is the single source of truth — the upload script regenerates the catalog `.md` from it.
  Manual edits to the `.md` file will be overwritten on the next upload.
- Re-running the upload is safe — files are upserted (overwritten, not duplicated).

## Sending emails to participants

Available templates: `welcome`, `welcome-lv`, `course-update`

```bash
# Preview without sending
npm run send-email -- <slug> <template> --dry-run

# Send to a single address (for testing)
npm run send-email -- <slug> <template> --to your@email.com

# Send to all course participants
npm run send-email -- <slug> <template>
```

### Examples

```bash
npm run send-email -- is-auditor welcome-lv --dry-run
npm run send-email -- is-auditor welcome-lv --to test@example.com
npm run send-email -- is-auditor course-update
```

## Adding a new course

1. Create `course-source/<new-slug>/` directory
2. Copy `scripts/meta.example.yaml` to `course-source/<new-slug>/meta.yaml` and fill it in
3. Add HTML lesson files to the directory
4. Run `npm run upload-course -- <new-slug>`
5. Commit the generated `content/courses/<new-slug>.md`
